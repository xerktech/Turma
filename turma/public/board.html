Each host agent polls its org's tracker — Jira Cloud (JIRA_SITE / JIRA_EMAIL / JIRA_TOKEN) or a self-hosted/cloud Azure DevOps board (AZDO_URL / AZDO_TOKEN) in the agent's env — for tickets assigned to that user, active work plus the last two weeks of Done, and the board unifies every host's report into one Kanban (hosts sharing an org collapse into one board, deduped by issue key). Columns follow the tracker's universal status categories, with an In Review column carved out for review/testing statuses; each card's pill shows the org's own status name. Pull-only: nothing here writes back to the tracker.
