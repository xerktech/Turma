Each host agent polls its org's Jira Cloud with user-scoped creds (JIRA_SITE / JIRA_EMAIL / JIRA_TOKEN in the agent's env) for tickets assigned to that user — active work plus the last two weeks of Done — and the board unifies every host's report into one Kanban (hosts sharing an org collapse into one board, deduped by issue key). Columns are Jira's universal status categories; each card's pill shows the org's own status name. Pull-only: nothing here writes to Jira.
